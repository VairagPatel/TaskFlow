🎯 DEPLOYMENT STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: INITIALIZE GIT (1 minute)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Open terminal in taskflow-final folder
□ Run: git init
□ Run: git add .
□ Run: git commit -m "Initial commit: TaskFlow Team Task Manager"
□ Verify: See commit success message

STEP 2: CREATE GITHUB REPOSITORY (2 minutes)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Go to: https://github.com/new
□ Repository name: taskflow-team-manager
□ Visibility: PUBLIC ⚠️
□ DO NOT add README, .gitignore, or license
□ Click "Create repository"
□ Keep the page open (you'll need the commands)

STEP 3: PUSH TO GITHUB (1 minute)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Copy commands from GitHub page (section: "push an existing repository")
□ Paste in terminal:
  git remote add origin https://github.com/YOUR_USERNAME/taskflow-team-manager.git
  git branch -M main
  git push -u origin main
□ Enter credentials if asked (use Personal Access Token, not password)
□ Verify: See "Branch main set up to track remote branch main"

STEP 4: DEPLOY TO RAILWAY (5 minutes)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Go to: https://railway.app
□ Click "Login" → "Login with GitHub"
□ Authorize Railway
□ Click "New Project"
□ Select "Deploy from GitHub repo"
□ Choose "taskflow-team-manager"
□ Click "Deploy Now"
□ Wait for build to complete (watch logs)
□ Verify: See "Deployment successful" or "Success" status

STEP 5: GENERATE DOMAIN (1 minute)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
□ Click on your deployment
□ Go to "Settings" tab
□ Scroll to "Networking" section
□ Click "Generate Domain"
□ Copy the URL (e.g., taskflow-production.up.railway.app)
□ Save URL: _________________________________

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

→ Use Personal Access Token from: https://github.com/settings/tokens

Problem: Railway build fails
→ Check logs, ensure requirements.txt is correct, redeploy

Problem: 502 Bad Gateway
→ Wait 2-3 minutes, check logs, redeploy if needed

Problem: Can't login
→ Check logs for seed completion, redeploy to regenerate data
